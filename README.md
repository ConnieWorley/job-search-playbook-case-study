# Job Search Playbook — Business Process Case Study

A case study in process analysis, redesign, and AI-integrated solution design — built and documented end-to-end as a portfolio project.

## The Problem

Job searching is a broken process. Most people apply, wait, and wonder — with no consistent method, no visibility into where things stand, and no system connecting the dozens of small tasks a search actually requires (tailoring materials, tracking outreach, prepping for interviews). The result is a process that depends entirely on memory and willpower, and breaks down under stress at exactly the moment it matters most.

This project treats the job search itself as a business process: something that can be observed, diagnosed, and redesigned — not just a personal to-do list.

## The Approach

The work followed a standard process-improvement structure:

1. **Current-state mapping** — documenting the job search as it actually happens today, lane by lane, to surface where it breaks down.
2. **Root cause and gap analysis** — identifying which breakdowns were tooling gaps versus process gaps.
3. **Future-state redesign** — rebuilding the process stage by stage, with clear ownership of each task (job seeker, application, or AI-assisted step).
4. **Stakeholder and role definition** — assigning explicit roles (Process Analyst, UX Designer, Product Owner, and others) to keep the build accountable, even as a solo project.
5. **Translation into product** — converting the redesigned process into screens, wireframes, and a working application.

## What's in This Repository

- **BPMN Current State** — the job search process as it exists without a system: manual, inconsistent, and full of gaps.
- **BPMN Future State (Stages 0–5)** — the redesigned process, broken into stages from setup through interview prep, showing exactly what changed and why at each step.
- **Stakeholder Register** — the roles behind the project and what each one is responsible for.
- **Screen Map** — the application's navigation structure and information architecture.
- **Annotated Layout Spec** — detailed, screen-by-screen behavioral and design specifications.
- **Wireframes** — visual mockups of the core screens.

## Why This Matters

The value of this project isn't the app itself — it's the discipline behind it. Every redesign decision traces back to a documented breakdown in the current-state process. Every screen traces back to a documented user need. That traceability, from diagnosed problem to shipped solution, is the core skill this case study is meant to demonstrate.
